# Contributing

Thanks for your interest in contributing to One-For-All.

The goal of this project is to build practical, accessible resources for early-stage founders and startup builders.

## What kind of contributions are welcome

You can contribute by:

- Improving existing files
- Fixing clarity, grammar, or structure
- Suggesting better resources
- Adding practical frameworks, guides, or checklists
- Sharing templates that help early-stage builders think and execute better

## Contribution principles

Please keep contributions:

- Clear
- Practical
- Beginner-friendly
- Well-structured
- Focused on usefulness over jargon

## How to contribute

1. Fork the repository
2. Create a new branch
3. Make your changes
4. Submit a pull request with a clear explanation

## Before submitting

Ask:

- Is this useful for an early-stage founder?
- Is it easy to understand?
- Is it actionable?
- Is it better than vague startup advice?

## Notes

Small improvements are welcome. This project is early and will improve through consistent iteration.
